## structure

### adapter

继承一个接口，然后重写里面的方法，这样的话，就可以让任意的对象，都实现这个功能了

继承接口的话，就是实现了适配器模式

### bridge

构造器可以传入一个对象，通过这个对象的多态，实现不同的行为

例如：每个页面切换不同的主题

### composite

将对象组合为树形结构

* [ ] 真的这么简单吗

### decorator

构造器传入同样类型的对象，实现运行时动态更改行为

例如：买奶茶添加很多东西，最后一并结账

### facade 外观

为复杂的对象提供了简化的操作接口

### flyweight

如果没有的话，就创造对象，有的话就直接使用

* [ ] 会不会有互相影响的问题

### proxy

全部重写方法，可以加一些额外逻辑
